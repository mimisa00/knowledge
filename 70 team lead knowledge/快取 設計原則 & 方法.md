### Cache 使用目的(優點)
1. 保護資源不被大量攻擊/占用 ( 如 DB SQL 資料查詢 cache, rest 資料 Cache, 以免大量存取造成資源 crash )
2. 提升系統效率, 消弭因資源連線存取或大量計算的時間耗費. ( 如 client cache 降低遠端存取時間, or 非即時複雜金額運算資料 cache 的計算時間 )
### 非因上述目的時應更謹慎使用 Cache. 以免沒好處反帶來副作用 (缺點)
1. 資料不即時造成作業異常甚至資料錯亂
2. 增加維護成本 (增加 cache 設定, 需手動清 cache…. )
### Cache 的使用應仔細觀察
1. 設計. 以了解資料存取頻率
2. 可允許非即時期限
3. 資料更新頻率等
### 每個資料 cache 應該都要有細部控制, 不應一體適用.
( e.g. 每個資料 cache 都是 30mins after access ) 細節亦應與相關人員討論在實際生產環境是否會有錯亂問題.
