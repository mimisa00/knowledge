## 版號定義  A.B.C
1. 版本號碼分列至三個位置(A.B.C)，各位置數字則代表每次更版目的，確定更版目的位置時，則對該位置進行加1動作以決定上線版號。
2. 例如：當前版號為 1.0.0 此次更版目的為 B 則更版後版號為 1.1.0
3. 版號位置目的定義
> * A = 大改版
> * B = 需求改版
> * C = 問題修正改版
4. 版號權重比較 A > B > C
> * A可含 B+C
> * B可含 C
5. 問題修正改版增加版號時機
> * 正式區發生無法操作，需臨時更版修正
> * 維護時間無其它目的B項目

## 分支建立定義
- default (single)
  *  定義 : lib code 最後合併位置 (lib code 泛指無法獨立執行服務之程式碼，其作用為提供各專案程式碼共用用途之程式)
  *  建立 : 版控系統預設branch，故無法自行建立
  *  名稱 : NONE
  * 備註 : 有些 lib code 最後是合併回 testing
- issue or feature (multi)
  * 定義 : 此分支對應 JIRA project 所開立 issue
  * 建立 : from version branch
  * 名稱 : 分支名稱以 JIRA issue 代號加序列為主 (不要把title加入)
  * 例 PROJECTA-001、PROJECTB-002
  *備註 : NONE
- version (multi)
  * 定義 : 參考版號定義 A.B.C
  * 建立 : from testing branch or default branch (lib)
  * 名稱 : 參考版號定義 A.B.C
  * 備註 : NONE
- testing (single)
  * 定義 : 工程師整合測試(SIT SERVER)、程式碼定版
  * 建立 : when create scm repository 
  * 名稱 : 固定不變
  * 備註 : NONE
- release (single)
  * 定義 : 正式區部署用，合併 testing UAT TAG
  * 建立 : when create scm repository 
  * 名稱 : 固定不變
  * 備註 : NONE
- TAG (multi)
  * 定義 : 標記 testing 部署錨點、標記 release 部署錨點
  * 建立 : 單次循環維護週期，程式碼確定不再變更
  * 名稱 : 
  * QAT  = QA 測試部署 testing 標記錨點 (測試區)
  * UAT  = OP 測試部署 testing 標記錨點 (測試區)
  * RL   = OP 測試部署 release 標記錨點 (正式區)
  * PRL  = OP 測試部署 release 前回維護標記錨點 (正式區)
